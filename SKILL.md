# Daily Review — ежедневный анализ и улучшения

## Goal
Прочитать все сессионные логи за день, найти паттерны и проблемы, предложить конкретные улучшения для CLAUDE.md/скиллов/воркфлоу, создать скомпанованный дневной лог, архивировать обработанные сессии.

## Paths
- Сессии: `~/.claude/sessions/session-*.md`
- Дневные логи: `~/.claude/sessions/daily/daily-{YYYY-MM-DD}.md`
- Архив: `~/.claude/sessions/archive/`
- CLAUDE.md: `~/.claude/CLAUDE.md`
- Memory: `~/.claude/projects/memory/` (автоопределяется по текущему проекту)
- Паттерны: `~/.claude/projects/memory/patterns.md`

## Process

### Step 1: Сбор сессий
1. Прочитать все файлы `~/.claude/sessions/session-{today}*.md`
2. Если нет файлов за сегодня — сказать и предложить посмотреть за другую дату
3. Показать краткий список: сколько сессий, их темы

### Step 2: Анализ паттернов
Прочитать все сессии и найти:

**Проблемы и трение:**
- Где буксовали? Что пришлось переделывать?
- Какие ошибки повторялись?
- Где контекст терялся или деградировал?
- Какие инструменты/скиллы не сработали как ожидалось?

**Что сработало:**
- Удачные решения и подходы
- Эффективные паттерны работы
- Скиллы/инструменты которые хорошо отработали

**Метрики (если можно извлечь):**
- Количество сессий
- Основные темы/проекты
- Соотношение "делал" vs "чинил/переделывал"

### Step 2.5: Легкий чек системы
Быстрая проверка здоровья системы (каждый день):

1. **CLAUDE.md** — посчитать строки. Если > 100 строк — флагнуть "CLAUDE.md растет, подумай о выносе в docs/"
2. **Voice DNA** — проверить `~/.claude/docs/voice-dna.md`. Есть ли обновления за сегодня? Cringe List не переполнен?
3. **Memory файлы** — `~/.claude/projects/memory/*.md` (автоопределяется). Когда последний раз обновлялись? Если > 7 дней — флагнуть
4. **Скиллы** — `~/.claude/skills/*/SKILL.md`. Были ли использованы за последнюю неделю (по сессиям)? Если скилл ни разу не упоминался — отметить
5. **Противоречия** — быстрый скан CLAUDE.md на конфликтующие правила (например, два правила которые говорят делать по-разному)

Формат вывода для этого шага:
```
## Здоровье системы
- CLAUDE.md: {N} строк {ok / внимание: растет}
- Memory: обновлялась {дата} {ok / устарела}
- Скиллы: {N} активных, {M} не использовались
- Проблемы: {список или "нет"}
```

**По пятницам** — добавить напоминание:
```
🔍 Пятница — время полного аудита системы!
Рекомендую пройтись по:
1. CLAUDE.md — актуальны ли все правила? Что убрать/добавить?
2. Voice DNA — отражает ли текущий стиль?
3. Скиллы — все работают? Нужны ли новые?
4. Memory — нет ли протухшей информации?
5. Архив сессий — можно ли почистить старше 30 дней?

Запустить полный аудит? (да/нет)
```

Если пользователь говорит "да" — пройтись по каждому пункту детально, прочитать файлы, предложить конкретные правки.

### Step 3: Предложения по улучшению
На основе анализа — конкретные предложения. Каждое должно быть:
- **Конкретным** — не "улучшить воркфлоу", а "добавить правило X в CLAUDE.md секцию Y"
- **Применимым** — можно сделать прямо сейчас
- **Обоснованным** — ссылка на конкретную проблему из сессии

Категории предложений:
1. **CLAUDE.md** — новые правила, уточнения существующих
2. **Скиллы** — улучшения, новые скиллы
3. **Memory** — что добавить в patterns.md
4. **Воркфлоу** — как изменить процесс работы

### Step 4: Показать пользователю
Формат вывода:

```
## Обзор дня — {дата}
{кол-во} сессий: {список тем через запятую}

## Что шло хорошо
- ...

## Где буксовали
- ...

## Предложения по улучшению
### 1. {предложение}
- Проблема: {что случилось}
- Решение: {что изменить}
- Где: {CLAUDE.md / скилл / память}

### 2. ...

## Применить?
```

### Step 5: Применение (с подтверждением)
Спросить пользователя какие предложения применить. Для каждого одобренного:
- Внести изменения в соответствующий файл
- Показать diff

### Step 6: Дневной лог
Создать `~/.claude/sessions/daily/daily-{YYYY-MM-DD}.md`:

```markdown
# Дневной лог — {дата}

## Сессии
{для каждой сессии — тема + 1-2 строки ключевое}

## Ключевые решения дня
{что решили, что изменили}

## Результаты
{что создали, что починили, что улучшили}

## Применённые улучшения
{какие изменения в CLAUDE.md/скиллах/памяти}

## Заметки для недельного дневника
{3-5 пунктов — самое важное за день, пригодится при составлении недельного обзора}
```

### Step 7: Ротация
После создания дневного лога:
1. Переместить обработанные session-*.md в `~/.claude/sessions/archive/`
2. НЕ удалять — переместить. Удаление только если архив старше 30 дней
3. Показать: "Архивировано N сессий. Дневной лог: {путь}"

## Правила
- Никогда не удалять сессионные логи без создания дневного лога
- Дневной лог создается ВСЕГДА, даже если нет предложений по улучшению
- Предложения применяются ТОЛЬКО после подтверждения пользователя
- Если пользователь закинул заметки из claude.ai — учитывать их в анализе
- Фокус анализа: только улучшения работы с Claude Code (не продуктивность в целом)

## Weekly Audit (по пятницам)
По пятницам Step 2.5 расширяется до полного аудита с напоминанием. Это не отдельный скилл — просто более глубокая версия ежедневного чека. Определение дня: `date +%u` — если 5, значит пятница.

## Weekly Digest
Дневные логи в `daily/` накапливаются для недельных дневников. Когда будет 7+ дневных логов — можно предложить пользователю сгенерировать недельный обзор.
